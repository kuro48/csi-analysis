┌─────────────┐
│ フェーズ3   │  Witness生成
└─────────────┘
              │
              ▼
   ┌──────────────────────────────────────────┐
   │  Witness計算モジュール (JavaScript)       │
   │                                          │
   │  ┌────────────────────────────────────┐  │
   │  │ ステップ1: ベクトル準備             │  │
   │  │  • 参照ベクトル (4次元)            │  │
   │  │  • 候補ベクトル群 (2個 × 4次元)    │  │
   │  └────────────────────────────────────┘  │
   │  ┌────────────────────────────────────┐  │
   │  │ ステップ2: 数学的計算               │  │
   │  │                                    │  │
   │  │  normA = √(Σ Ai²)                 │  │
   │  │        = √(8500² + 6200² + ...)    │  │
   │  │        = 11747                     │  │
   │  │                                    │  │
   │  │  dotProduct = Σ(Ai × Bi)          │  │
   │  │             = 8500×8200 + ...      │  │
   │  │             = 138,080,000          │  │
   │  │                                    │  │
   │  │  similarity = (A·B × 10⁴)         │  │
   │  │             / (||A|| × ||B||)      │  │
   │  │             = 9990                 │  │
   │  │             ≈ 0.999                │  │
   │  └────────────────────────────────────┘  │
   │  ┌────────────────────────────────────┐  │
   │  │ ステップ3: Witness JSON生成         │  │
   │  └────────────────────────────────────┘  │
   └──────────────────┬───────────────────────┘
                      │
                      ▼
   ┌───────────────────────────────────────┐
   │  Witness JSON                         │
   │  {                                    │
   │    "reference": [8500, 6200, ...],    │
   │    "candidates": [[8200, ...], ...],  │
   │    "referenceNorm": 11747,            │
   │    "candidateNorms": [11831, 11489],  │
   │    "similarities": [9990, 8281]       │
   │  }                                    │
   └───────────────────┬───────────────────┘
                       │

┌─────────────┐
│ フェーズ4   │  ZKP回路処理
└─────────────┘
                       │
                       ▼
   ┌────────────────────────────────────────────────┐
   │  Circom回路 (csi_subcarrier_selector.circom)   │
   │                                                │
   │  ┌──────────────────────────────────────────┐ │
   │  │ 1. DotProduct(N=4)                       │ │
   │  │    products[i] = vectorA[i] × vectorB[i] │ │
   │  │    dotProduct = Σ products               │ │
   │  │                                          │ │
   │  │    制約数: 4乗算 + 3加算 = 7制約         │ │
   │  └──────────────────────────────────────────┘ │
   │  ┌──────────────────────────────────────────┐ │
   │  │ 2. VectorNorm(N=4)                       │ │
   │  │    squares[i] = vector[i] × vector[i]    │ │
   │  │    normSquared = Σ squares               │ │
   │  │                                          │ │
   │  │    制約数: 4乗算 + 3加算 = 7制約         │ │
   │  └──────────────────────────────────────────┘ │
   │  ┌──────────────────────────────────────────┐ │
   │  │ 3. CosineSimilarityVerified              │ │
   │  │    検証制約:                             │ │
   │  │    similarity × normA × normB            │ │
   │  │    === dotProduct × SCALE                │ │
   │  │                                          │ │
   │  │    lhs = 9990 × 11747 × 11831           │ │
   │  │    rhs = 138080000 × 10000              │ │
   │  │    lhs === rhs  ✓                        │ │
   │  │                                          │ │
   │  │    制約数: 3乗算 + 1等価性 = 4制約       │ │
   │  └──────────────────────────────────────────┘ │
   │  ┌──────────────────────────────────────────┐ │
   │  │ 4. SubcarrierSelector                    │ │
   │  │    閾値チェック: similarity >= 7000      │ │
   │  │    最大値選択: max(similarities)         │ │
   │  │    出力: bestIndex, bestSimilarity       │ │
   │  │                                          │ │
   │  │    制約数: 10-15制約                     │ │
   │  └──────────────────────────────────────────┘ │
   │                                                │
   │  総制約数: 約50-60制約                         │
   └────────────────────┬───────────────────────────┘
                        │ R1CS制約システム

## パフォーマンス分析

### 各フェーズの処理時間

| フェーズ | 処理内容 | 処理時間 | 累積時間 |
|---------|---------|---------|---------|
| 1. データ取得 | エッジデバイス→サーバー (HTTP) | ~50ms | 50ms |
| 2. 前処理 | ノイズ除去、正規化、変換 | ~10ms | 60ms |
| 3. Witness生成 | 数学的計算 (JS) | ~20ms | 80ms |
| 4. ZKP回路処理 | R1CS制約生成 | ~5ms | 85ms |
| 5. 証明生成 | Groth16証明 | ~100ms | 185ms |
| 6. 検証 | ペアリング演算 | ~10ms | 195ms |
| 7. 保存・配信 | DB保存、WebSocket配信 | ~5ms | 200ms |

**合計レイテンシ: ~200ms (エンドツーエンド)**

### ボトルネック分析

```
証明生成 (100ms) ████████████████████████████████████████ 51%
データ取得 (50ms) ████████████████████ 26%
Witness生成 (20ms) ████████ 10%
検証 (10ms) ████ 5%
前処理 (10ms) ████ 5%
その他 (10ms) ████ 5%
```

**最適化ポイント**:
1. 証明生成の並列化 (複数候補を並列処理)
2. Witness計算のキャッシング (参照ベクトルのノルム)
3. ネットワーク遅延の削減 (エッジ→サーバー)

## メモリ使用量

| コンポーネント | メモリ使用量 | 説明 |
|--------------|-------------|------|
| 生CSIデータ | ~1KB | 56サブキャリア × 1フレーム |
| Witness JSON | ~500B | 4次元ベクトル × 3個 |
| R1CS制約 | ~50KB | 約50-60制約 |
| 証明データ | ~500B | Groth16証明 (3要素) |
| 検証キー | ~2KB | 一度のみロード |

**総メモリ: ~55KB (ピーク時)**

## セキュリティとプライバシー

### ゼロ知識証明による保護

```
┌─────────────────────────────────────────────────────┐
│  プライバシー保護されたデータフロー                  │
└─────────────────────────────────────────────────────┘

公開される情報:
  • 選択されたサブキャリアのインデックス (0 or 1)
  • 類似度の値 (0.999)
  • ZKP証明データ

秘匿される情報:
  ✓ 生CSI振幅・位相データ
  ✓ 正規化前のベクトル値
  ✓ 内積の中間計算値
  ✓ ノルムの中間計算値
  ✓ 候補の詳細な特徴

検証可能な性質:
  ✓ 選択が正しく行われたこと
  ✓ 類似度計算が正確であること
  ✓ 閾値判定が適切であること
  ✓ アルゴリズムが改ざんされていないこと
```

### 信頼モデル

```
セットアップフェーズ:
┌──────────────────────┐
│ Powers of Tau        │  信頼されたセットアップ
│ (公開パラメータ)      │  多者間計算により生成
└──────────────────────┘
         │
         ▼
┌──────────────────────┐
│ 回路固有キー生成      │  回路ごとに1回のみ
│ (zkey)               │
└──────────────────────┘

実行フェーズ:
┌──────────────────────┐
│ Prover (証明者)       │  エッジデバイス / サーバー
│ • Witness計算        │  生データにアクセス可能
│ • 証明生成           │
└──────────────────────┘
         │
         ▼
┌──────────────────────┐
│ Verifier (検証者)     │  フロントエンド / 第三者
│ • 証明検証のみ       │  生データにアクセス不可
│ • 公開入力のみ使用   │
└──────────────────────┘
```

## スケーラビリティ

### 並列処理の可能性

```
シナリオ1: 単一エッジデバイス
┌─────────┐
│ Device1 │──→ 証明生成 (100ms)
└─────────┘

シナリオ2: 複数エッジデバイス (並列)
┌─────────┐
│ Device1 │──→ 証明生成 (100ms) ┐
└─────────┘                     │
┌─────────┐                     ├→ 並列検証 (10ms)
│ Device2 │──→ 証明生成 (100ms) │
└─────────┘                     │
┌─────────┐                     │
│ Device3 │──→ 証明生成 (100ms) ┘
└─────────┘

スループット: 3デバイス × 10選択/秒 = 30選択/秒
```

### バッチ処理

```
通常処理:
選択1 (200ms) → 選択2 (200ms) → 選択3 (200ms)
合計: 600ms

バッチ処理:
┌─────────┬─────────┬─────────┐
│ 選択1   │ 選択2   │ 選択3   │ ← 並列Witness生成 (20ms)
└─────────┴─────────┴─────────┘
         ↓
    一括証明生成 (150ms)
         ↓
      一括検証 (15ms)

合計: 185ms (68%削減)
```
