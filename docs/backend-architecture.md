# バックエンドアーキテクチャ詳細

CSI Web Platform バックエンドの各ファイル・ディレクトリの役割を詳細解説

---

## 📁 ディレクトリ構造概要

```
backend/
├── app/                    # メインアプリケーション
│   ├── main.py            # アプリケーションエントリーポイント
│   ├── core/              # コア設定・基盤機能
│   ├── api/               # APIルーティング・エンドポイント
│   ├── models/            # SQLAlchemyデータモデル
│   ├── schemas/           # Pydanticスキーマ定義
│   ├── services/          # ビジネスロジックサービス
│   └── tasks/             # バックグラウンドタスク
├── database/              # データベースマイグレーション
├── alembic/               # Alembicマイグレーション管理
└── uploads/               # アップロードファイル保存
```

## 🔧 アプリケーション基盤層

### **app/main.py** - アプリケーションエントリーポイント
**役割**: FastAPIアプリケーションの起動・初期化・設定統合

**主要機能**:
- FastAPIアプリケーション作成と設定
- CORS設定（フロントエンド・エッジデバイスアクセス許可）
- APIルーター統合（`/api/v2`プレフィックス）
- 起動/終了イベント処理（タスクキュー接続・切断）
- 基本エンドポイント提供（`/`, `/health`）

### **app/core/** - コア設定・基盤機能

#### **config.py** - アプリケーション設定管理
**役割**: 環境変数からアプリケーション設定を一元管理

**設定項目**:
- **基本設定**: プロジェクト名、バージョン、API prefix
- **データベース**: PostgreSQL接続情報
- **Redis**: キャッシュ・セッション管理
- **JWT**: 認証トークン設定
- **CORS**: 許可オリジン設定
- **IPFS**: 分散ストレージ連携
- **レート制限**: API呼び出し制限

#### **database.py** - データベース接続管理
**役割**: PostgreSQL接続とSQLAlchemy設定

**主要機能**:
- SQLAlchemyエンジン作成（コネクションプール設定）
- セッション作成（`SessionLocal`）
- ベースクラス定義（`Base`）
- 依存性注入用DB取得関数（`get_db`）
- データベース初期化関数（`init_db`）

#### **security.py** - セキュリティ・認証機能
**役割**: JWT認証とパスワード暗号化

**主要機能**:
- パスワードハッシュ化・検証
- JWTアクセストークン生成・検証
- トークンペイロード管理

#### **deps.py** - FastAPI依存性注入
**役割**: 認証・権限チェックの依存性提供

**主要機能**:
- 現在認証ユーザー取得
- 管理者権限チェック
- トークン検証・ユーザー特定

## 🛣️ APIルーティング層

### **app/api/routes.py** - APIルーター統合
**役割**: 各エンドポイントモジュールを `/api/v2` プレフィックスで統合

**統合エンドポイント**:
- `/health` - システム監視
- `/auth` - 認証系
- `/devices` - デバイス管理
- `/csi-data` - CSIデータ管理
- `/breathing-analysis` - 呼吸解析
- `/ws` - WebSocket通信
- `/ipfs` - IPFS連携
- `/tasks` - タスク管理

### **app/api/endpoints/** - 各機能別エンドポイント

#### **auth.py** - 認証系API
**役割**: ユーザー認証・セッション管理

**エンドポイント**:
- `POST /register` - ユーザー登録（重複チェック付き）
- `POST /login` - ログイン（JWTトークン発行）
- `POST /refresh` - トークン更新
- `POST /logout` - ログアウト
- `GET /me` - 現在ユーザー情報取得

#### **devices.py** - デバイス管理API
**役割**: エッジデバイスの登録・管理・監視

**主要機能**:
- デバイス登録・一覧・詳細・更新・削除
- デバイス状態監視（オンライン/オフライン）
- 所有者権限管理
- デバイス統計情報提供

#### **csi_data.py** - CSIデータ管理API
**役割**: CSIファイルのアップロード・管理・配信

**エンドポイント**:
- `POST /upload` - CSIファイルアップロード
- `GET /` - データ一覧（フィルタリング・ページネーション）
- `GET /{id}` - 特定データ詳細取得
- `DELETE /{id}` - データ削除
- `GET /stats/{device_id}` - デバイス統計

#### **breathing_analysis.py** - 呼吸解析API
**役割**: CSIデータ解析結果の管理・提供

**主要機能**:
- 解析結果一覧・詳細取得
- 統計情報提供
- 解析状態管理

#### **websocket.py** - リアルタイム通信
**役割**: WebSocket接続とリアルタイムデータ配信

**主要機能**:
- WebSocket接続管理
- デバイス状態更新配信
- 解析結果リアルタイム通知
- セッション状態通知

#### **ipfs.py** - IPFS連携API
**役割**: 分散ストレージとの連携管理

**主要機能**:
- IPFSファイルアップロード
- メタデータ管理
- ファイル永続化

#### **health.py** - システム監視API
**役割**: システム状態・依存関係の監視

**主要機能**:
- データベース接続状態チェック
- Redis接続状態チェック
- IPFS接続状態チェック
- 詳細ヘルスレポート

#### **tasks.py** - タスク管理API
**役割**: バックグラウンドタスクの管理・監視

**主要機能**:
- タスク状態確認
- タスクキュー監視
- タスク実行管理

## 🗄️ データモデル層

### **app/models/** - SQLAlchemyモデル定義

#### **base.py** - ベースモデル
**役割**: 全テーブル共通フィールドの定義

**共通フィールド**:
- `id` - UUID主キー
- `created_at` - 作成日時
- `updated_at` - 更新日時

#### **user.py** - ユーザーモデル
**役割**: システムユーザー情報管理

**テーブル**: `users`
**フィールド**:
- `username` - ユーザー名（一意）
- `email` - メールアドレス（一意）
- `password_hash` - パスワードハッシュ
- `is_active` - アクティブ状態
- `is_superuser` - 管理者権限
- `last_login_at` - 最終ログイン時刻

**リレーション**: デバイス所有関係（1:N）

#### **device.py** - デバイスモデル
**役割**: エッジデバイス情報・状態管理

**テーブル**: `devices`
**フィールド**:
- `device_id` - デバイス識別子（一意）
- `device_name` - デバイス名
- `device_type` - デバイス種別
- `location` - 設置場所
- `owner_id` - 所有者ID
- `is_active` - アクティブ状態
- `last_seen` - 最終接続時刻

**インデックス**: 検索最適化用複合インデックス
**リレーション**: ユーザー、CSIデータ、セッション、解析結果

#### **csi_data.py** - CSIデータモデル
**役割**: CSI測定データ・セッション管理

**テーブル**: `csi_data`, `sessions`

**CSIDataフィールド**:
- `device_id` - 測定デバイスID
- `session_id` - セッション識別子
- `raw_data` - 生CSIデータ（JSONB）
- `processed_data` - 処理済みデータ（JSONB）
- `file_path` - ファイル保存パス
- `file_size` - ファイルサイズ
- `ipfs_hash` - IPFS保存ハッシュ
- `status` - 処理状態

**Sessionフィールド**:
- `device_id` - 測定デバイスID
- `session_name` - セッション名
- `start_time` - 開始時刻
- `end_time` - 終了時刻
- `duration` - 継続時間（秒）
- `status` - セッション状態
- `meta_data` - メタデータ（JSONB）

**インデックス**: デバイス別・時系列検索最適化

#### **breathing_analysis.py** - 呼吸解析結果モデル
**役割**: CSIデータ解析結果・統計データ保存

**主要機能**:
- 解析アルゴリズム結果保存
- 呼吸パターン統計
- 解析精度メトリクス

## 🔗 システム間データフロー

### 1. CSIデータアップロードフロー
```
エッジデバイス → csi_data.py:upload_csi_data()
                ↓
            ファイル保存 (uploads/)
                ↓
            IPFS統合アップロード
                ↓
            PCAP解析実行 (pcap_analyzer.py)
                ↓
            PostgreSQL保存
                ↓
            WebSocket通知配信
                ↓
            フロントエンド更新
```

### 2. リアルタイム解析フロー
```
CSIデータ → realtime_csi_analyzer.py
            ↓
        呼吸パターン解析
            ↓
        breathing_analysis.py
            ↓
        WebSocket配信
            ↓
        フロントエンド表示
```

### 3. 認証・API呼び出しフロー
```
フロントエンド → auth.py (ログイン)
                ↓
            JWTトークン発行
                ↓
            各API呼び出し (deps.py認証)
                ↓
            サービス層処理
                ↓
            データベースアクセス
                ↓
            レスポンス返却
```

## 🏛️ アーキテクチャ設計原則

### レイヤードアーキテクチャ
```
API層 (endpoints/)
    ↓
サービス層 (services/)
    ↓
モデル層 (models/)
    ↓
データベース層 (PostgreSQL/Redis)
```

### 設計特徴
- **依存性注入**: FastAPI Depends活用
- **非同期処理**: async/await使用
- **型安全性**: Pydanticスキーマ検証
- **分離原則**: 関心事の分離
- **拡張性**: モジュール化・サービス分離
- **パフォーマンス**: Redis キャッシュ・インデックス最適化
- **リアルタイム**: WebSocket統合
- **分散ストレージ**: IPFS連携

このアーキテクチャにより、保守性・拡張性・パフォーマンスを確保した CSI 呼吸監視システムを実現しています。
