# 研究計画書

## 研究タイトル（案）

"Edge-Side Zero-Knowledge Proof for Privacy-Preserving Breathing Rate Monitoring via Wi-Fi CSI"

エッジデバイス上のゼロ知識証明を用いたWi-Fi CSIによるプライバシー保護型呼吸計測システム

---

## 研究概要

Wi-Fi CSI（Channel State Information）を用いた非接触呼吸監視において、生体データのプライバシー保護は未解決の課題である。既存研究は計測精度の向上に注力してきたが、計測データはサーバーに平文で送信・保存されており、クラウド側での情報漏洩リスクが存在する。

本研究では、エッジデバイス（Raspberry Pi + Nexmon）上でZKP（Zero-Knowledge Proof）証明を生成し、生のCSIデータをデバイス外に出すことなく呼吸レートの正当性を証明するシステムを提案する。

---

## システム構成

### ハードウェア
- **エッジデバイス**: Raspberry Pi + Nexmon CSI
- **Wi-Fi**: Broadcom NIC（Nexmon対応）、単一アンテナ
- **CSI仕様**: 最大256サブキャリア（802.11n）、amplitude + phase

### データフロー

```
[従来]
Raspberry Pi → (生CSIデータ送信) → クラウドサーバー → 解析・保存

[提案]
Raspberry Pi → CSI計測
            → 呼吸レート計算
            → ZKP証明生成（生データはデバイス外に出ない）
            → (証明 + 呼吸レートのみ送信) → ブロックチェーン記録
```

### ソフトウェアスタック
- **ZKP**: Circom + snarkjs（Groth16）
- **バックエンド**: FastAPI（Python）
- **フロントエンド**: Next.js + TypeScript
- **ブロックチェーン**: Ethereum（Ganache / 本番チェーン）
- **スマートコントラクト**: ZKProofRegistry.sol

## 脅威モデル

### 想定攻撃者：Semi-honest Cloud Server

クラウドサーバーはプロトコルには従うが、受け取った情報から可能な限り個人情報を引き出そうとする（正直だが好奇心旺盛なモデル）。

### 保護対象

| 情報 | リスク | 保護方法 |
|------|--------|---------|
| 生CSI波形 | 在室有無・移動パターン・環境構造の推定 | デバイス外に送信しない |
| 呼吸パターン（時系列） | 睡眠状態・ストレス状態の推定 | 呼吸レート（単一値）のみ公開 |
| 計測の事実 | 存在確認 | ブロックチェーン記録（改ざん防止のみ、匿名化は別途検討） |

### セキュリティ目標
- **Zero-Knowledge**: 証明からCSI生データを復元できない
- **Soundness**: 正しい計算を経ていない呼吸レートの証明は受理されない
- **Completeness**: 正しく計算された呼吸レートは必ず証明できる

## 技術的貢献

### 貢献1：呼吸レート計算のZKP回路設計（主貢献）

**課題**: 通常のFFTは浮動小数点演算を含みZKP回路化が困難

**解決策: 部分DFT（特定周波数ビンのみ計算）**

```
呼吸の周波数範囲: 0.1〜0.5 Hz（6〜30 bpm）
→ 全周波数帯域でなく、対象帯域の数十ビンのみDFTを計算
→ 整数近似・固定小数点演算で回路化可能
```

回路の入出力：
- **秘密入力**: CSIデータ（固定小数点化）
- **公開入力**: CSIデータのコミットメント（Poseidonハッシュ）
- **公開出力**: 呼吸レート（BPM）
- **証明内容**: 「この呼吸レートは、このハッシュに対応するCSIデータから正しく計算された」

### 貢献2：エッジデバイス上でのZKP実行評価

Raspberry Pi（リソース制約あり）上での証明生成の実現可能性を評価：
- 証明生成時間
- メモリ使用量
- CPU負荷・消費電力

### 貢献3：プライバシー保証の定量評価

**実証的アプローチ**:
1. 生CSIデータからの情報推定デモ（例：機械学習で在室/不在を分類可能であることを示す）
2. ZKP後の出力（呼吸レートのみ）からは同様の推定が不可能であることを示す
3. サーバーへの送信データ量の比較（CSI行列 数MB → 証明 数KB）

## 新規性の主張

既存の全論文との差別化：

| 観点 | 既存研究 | 本研究 |
|------|---------|--------|
| データ送信 | 生CSIデータをサーバーへ | 証明のみ送信 |
| プライバシーモデル | 未定義 | Semi-honest server モデル |
| 計測証明 | なし | ZKPによる正当性証明 |
| 改ざん防止 | なし | ブロックチェーン記録 |
| 実行場所 | サーバーサイド処理 | エッジデバイス上でZKP生成 |

## 未解決の課題・検討事項

1. **ZKP回路の制約数**: 部分DFTの制約数がRaspberry Piで許容範囲か（要プロトタイプ検証）
2. **固定小数点精度**: 整数近似による呼吸レート推定精度の劣化がどの程度か
3. **コミットメントスキーム**: CSIデータのハッシュ化にPoseidonを使うか（ZKP内で計算効率が良い）
4. **ブロックチェーン匿名性**: 証明記録者のウォレットアドレスと個人の紐付き問題
